## What's happening

Your `public/checkout.html` file contains a **hardcoded Anthropic API key** on line 9. This key is being sent directly from the browser to Anthropic's API. Even though this particular key is marked as a fake test fixture, in a real scenario this would mean:

- **Every user who visits your page downloads the key** in the JavaScript bundle
- An attacker can open DevTools, read the key, and use it to rack up charges on your Anthropic account
- Once shipped, the key is **permanently compromised** — even if you delete it from your repo, every user who already loaded the page has a copy

## Step 2: Understand the architecture change

Right now your flow is:
```
Browser → Anthropic API (with exposed key)
```

The secure flow must be:
```
Browser → Your Backend → Anthropic API (key stays server-side)
```

You need to create a server-side proxy endpoint that:
- Accepts requests from your frontend
- Adds the Anthropic key server-side (from environment variables)
- Forwards the request to Anthropic
- Returns the response to the browser

const Anthropic = require('@anthropic-ai/sdk');

exports.handler = async (event) => {
  if (event.httpMethod !== 'POST') {
    return { statusCode: 405, body: 'Method Not Allowed' };
  }

  const anthropic = new Anthropic({
    apiKey: process.env.ANTHROPIC_API_KEY,
  });

  try {
    const { message } = JSON.parse(event.body);
    
    const response = await anthropic.messages.create({
      model: 'claude-sonnet-4-5',
      max_tokens: 100,
      messages: [{ role: 'user', content: message || 'hi' }],
    });
    
    return {
      statusCode: 200,
      body: JSON.stringify(response),
    };
  } catch (error) {
    console.error(error);
    return {
      statusCode: 500,
      body: JSON.stringify({ error: 'Failed to get AI response' }),
    };
  }
};

## Why this matters

If an attacker gets your Anthropic key, they can:
- Run up unlimited charges on your account (Claude API calls cost money)
- Use your quota to DoS your legitimate users
- Exfiltrate any data you're sending to Claude (if you're processing user data)

By moving the key server-side, only your backend can use it. Even if an attacker reverse-engineers your frontend, they can't extract the key.
